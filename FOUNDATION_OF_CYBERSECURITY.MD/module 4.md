# Module 4: The Security Analyst Toolkit & Portfolios

## 1. Core Security Tools
As a security professional, these are the primary tools used to monitor, analyze, and defend networks:
* **SIEM (Security Information and Event Management):** An application that collects and analyzes log data to monitor critical activities (e.g., Splunk, Google Chronicle). It filters massive amounts of data to alert analysts to specific threats.
* **Network Protocol Analyzer (Packet Sniffer):** A tool designed to capture and analyze data traffic within a network (e.g., Wireshark). 
* **IDS (Intrusion Detection System):** An application that monitors system activity and network packets to alert teams of possible unauthorized access or attacks.
* **Antivirus/Anti-malware:** Software used to scan memory and prevent, detect, or eliminate malicious code.

## 2. Playbooks & Incident Response
**Playbook:** A manual providing step-by-step details for operational actions, especially during security incidents.
* **Chain of Custody Playbook:** Documents the possession and control of evidence during an investigation. It answers: who, what, where, and why evidence is held.
* **Protecting and Preserving Evidence Playbook:** Guides the handling of fragile/volatile digital data.
* **Order of Volatility:** The strict sequence outlining which data must be preserved first (data that disappears when a machine powers off is the highest priority).

## 3. Scripting & Operating Systems
* **Linux:** An open-source OS relying heavily on the command-line interface (CLI).
* **Python:** A programming language used by analysts to automate repetitive tasks and reduce human error.
* **SQL (Structured Query Language):** Used to create, interact with, and request information from databases.

## 4. Security Concepts
* **Web Vulnerability:** A flaw in a web app that allows threat actors to execute malicious code, steal data, or gain unauthorized access. (Tracked by OWASP Top 10).
* **Encryption:** Converting readable plaintext into secure ciphertext to ensure confidentiality. 
* **Penetration Testing (Pen Testing):** Simulated, authorized attacks to identify vulnerabilities in systems and networks before threat actors can exploit them.
